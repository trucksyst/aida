## Часть 2 — Рефакторинг Search UI (стиль DAT)

### 16. Объединение City + State в одно поле
**Файлы:** `ui/app.html`, `ui/app.js`
- Убраны отдельные поля State для Origin и Destination
- Формат ввода: `Chicago, IL` — парсится в `getSearchParams()` через `parseCityState()`
- Обновлены `updatePresetDisplay()` и `handlePresetSave()` — убрана проверка `originState`

### 17. Floating Labels (Material Design паттерн)
**Файлы:** `ui/app.html`, `ui/components/styles.css`
- Все input-ы search-панели используют floating label: label внутри поля как placeholder, при вводе/фокусе всплывает на верхний бордер
- CSS-селекторы: `:focus + label`, `:not(:placeholder-shown) + label`
- HTML: `<input placeholder=" ">` + `<label for="...">` после input
- Floated state: 9px, uppercase, `background: var(--bg-secondary)`, `z-index: 1`
- Date, Type, Saved Searches — label всегда в floated позиции (всегда есть значение)

### 18. Убраны вертикальные разделители
**Файлы:** `ui/app.html`, `ui/components/styles.css`
- Удалены все `<div class="search-divider">` из HTML
- Удалено CSS-правило `.search-divider`

### 19. Добавлен Destination Radius
**Файлы:** `ui/app.html`, `ui/app.js`
- Новое поле `dest-radius` (default: 150, min: 0, max: 500)
- Добавлен в `getSearchParams()` как `destRadius`
- Добавлен в live preview и Enter-key listeners

### 20. Dropdown-ы → position:fixed (выход из overflow:hidden)
**Файлы:** `ui/components/styles.css`, `ui/app.js`
- `equip-dropdown`, `location-autocomplete`, `preset-dropdown` переведены на `position: fixed; z-index: 9999`
- JS: позиционирование через `getBoundingClientRect()` при каждом открытии
- `equip-dropdown` перемещён в `document.body` через `appendChild()` при инициализации
- Close handler обновлён: проверяет `#equip-multi` и `#equip-dropdown`
