# AIDA — если «не работает», что прислать

**Работа с AIDA — только один профиль браузера.** Используй один Chrome (с расширением): открой one.dat.com, войди, сделай поиск, открой вкладку AIDA. Не запускай одновременно «Google Chrome for Testing» (скрипт open-dat) для повседневной работы — он только для тестов.

Я не могу подключиться к твоему браузеру. Чтобы понять причину за один раз:

## Где смотреть логи (каждый шаг пишется в консоль)

| Что смотреть | Где открыть консоль |
|--------------|----------------------|
| **Харвестер** (перехват на one.dat.com) | Вкладка **one.dat.com** → F12 → вкладка Console. Ищи `[AIDA/Harvester]` — загрузка скрипта, токен, поиск, отправка DAT_SEARCH_RESPONSE. |
| **Core (background)** | `chrome://extensions` → AIDA → **Service worker**. Ищи `[AIDA/Core]` — входящие сообщения, handleDatSearchResponse, GET_LOADS, setLoads. |
| **UI (вкладка AIDA)** | Вкладка **AIDA** → F12 → Console. Ищи `[AIDA/UI]` — init, GET_LOADS, doSearch, storage work:loads changed. |

Фильтр в консоли: введи `AIDA` или `[AIDA/`, чтобы видеть только наши логи.

## Если UI не открывается по клику на иконку

1. **Перезагрузи расширение**: chrome://extensions → AIDA → кнопка **Обновить** (после правок в коде).
2. **Иконка в панели**: нажми на значок пазла (расширения) в тулбаре → найди AIDA → нажми булавку, чтобы закрепить иконку AIDA. Потом кликай по иконке AIDA.
3. **Ошибка при загрузке**: chrome://extensions → AIDA → ссылка **Service worker**. Если видишь красные ошибки в консоли — background не загрузился, из‑за этого клик не срабатывает. Скопируй текст ошибки и пришли.
4. **Открыть UI вручную**: в адресной строке введи `chrome-extension://ВАШ_ID_расширения/ui/app.html` (ID смотри на chrome://extensions у карточки AIDA — «ID: xxxxx») и нажми Enter — откроется интерфейс AIDA.

## 1. Открой консоль Service Worker

- `chrome://extensions` → найди **AIDA** → нажми **Service worker** (или «Background page»).
- Откроется консоль именно расширения (не страницы).

## 2. Воспроизведи проблему

- Открой вкладку **one.dat.com**, войди, сделай поиск (чтобы мог подхватиться токен).
- Открой вкладку **AIDA**, нажми **Search**.

## 3. Скопируй только строки AIDA

В консоли Service Worker отфильтруй или найди строки, где есть **`[AIDA/`**:

- `[AIDA/Core] tokens: DAT=... Truckstop=...`
- `[AIDA/Core] DAT_SEARCH_RESPONSE: N raw → M normalized` — перехват поиска со страницы DAT (если N=0, харвестер не видит ответ; если M=0, формат ответа другой)
- `[AIDA/Core] searchLoads: ...`
- `[AIDA/Core] Found ... loads`
- `[AIDA/DAT] ...` или `[AIDA/Core] ... adapter error`

Скопируй **эти строки** (5–15 штук) и вставь в чат. По ним можно будет сказать: нет токена, ответ не перехватывается, ошибка API, пустой ответ и т.д.

## 4. Если токен DAT = NO

Значит харвестер не перехватил токен. Проверь:

- Вкладка one.dat.com открыта в **том же** профиле Chrome, что и расширение.
- На one.dat.com ты **залогинен** и сделал поиск (не просто открыл страницу).
- Расширение **перезагружено** после правок (на chrome://extensions кнопка «Обновить» у AIDA).

После этого снова открой one.dat.com, сделай поиск, потом во вкладке AIDA нажми Search и пришли новые строки из консоли с `[AIDA/`.

## 5. Если грузы на DAT есть, а в AIDA пусто

1. **Перезагрузи расширение**: chrome://extensions → AIDA → кнопка «Обновить».
2. **Перезагрузи вкладку one.dat.com** (F5 или Ctrl+R), чтобы харвестер снова подхватил страницу.
3. На one.dat.com снова сделай **поиск** (тот же или новый).
4. Открой консоль Service Worker и посмотри, появилась ли строка `[AIDA/Core] DAT_SEARCH_RESPONSE: N raw → M normalized`. Если да — грузы ушли в Storage; открой вкладку AIDA и нажми Search или обнови её. Если такой строки нет — харвестер не перехватил ответ (другой URL или формат запроса).
