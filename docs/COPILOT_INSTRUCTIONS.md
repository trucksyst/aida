# Copilot инструкции для AIDA

Кастомные инструкции для работы с проектом AIDA.

---

## О проекте

**AIDA** — Chrome расширение (Manifest v3) для диспетчеров грузовиков. Агрегирует доступные грузы со всех крупных лоадбордов:
- **DAT.com** (DAT)
- **Truckstop.com** (Truckstop)
- **TruckerPath.com** (TruckerPath/tp)
- **123Loadboard.com** (123LB)

### Архитектура

- `background/` — Service Worker (центральный оркестратор, registry адаптеров, управление API)
- `ui/` — интерфейс расширения (HTML, CSS, Leaflet.js для карт)
- `adapters/` — плагины для каждого борда (стандартный контракт)
- `auth/` — модули авторизации для каждого адаптера
- `harvesters/` — сборщики данных (интеграция с Retell AI, TruckerPath)

### Технологии

JavaScript (ES modules), Service Worker API, Leaflet.js, Retell AI, OpenStreetMap/OSRM.

- Только код остаётся как есть (JavaScript/CSS без перевода)

- Избегайте лишних вступлений

- Вопросы о том, как что-то работает
- Проблемы в одном файле

### Когда показывать план

### Когда редактировать напрямую
- Исправление очевидных ошибок
- Добавление комментариев/документации
- После одобрения плана пользователем

## Архитектурные правила

### Адаптеры
- Стандартный контракт для каждого лоадборда
- Auth модули отделены (в `auth/`)
- API-детали остаются внутри адаптера
- Результаты в общем формате

### Service Worker (`background/`)
- `background.js` — инициализация, registry адаптеров
- `retell.js` — интеграция с Retell AI
- `storage.js` — работа с localStorage/IndexedDB
- Нет доступа к DOM, работает через message API

### UI (`ui/`)
- `app.html` — структура
- `app.js` — логика, обработчики
- `components/styles.css` — стили
- `lib/leaflet.js` — карты
- `data/locations.js` — данные о локациях

### Конвенции кода
- ES modules (`import`/`export`)
- Async/await, Promise
- Vanilla JS, без jQuery
- Помнить о Chrome Extension ограничениях (CSP, manifest v3)

### Проверки перед добавлением
- Manifest.json обновлён при новых permissions
- Service Worker может быть выгружен в любой момент → используйте storage
- Новый адаптер проверен на контракт с registry

## Анализ кода

При анализе кода AIDA:
1. Определите компонент (adapter, auth, harvester, UI, background)
2. Посмотрите контракт между компонентами
3. Проследите цепь вызовов: UI → background message → adapter → API
4. Учтите асинхронность (Promise-based, async/await)

Предлагайте решение в контексте существующей архитектуры. Не предлагайте кардинальные переделки без обсуждения.
